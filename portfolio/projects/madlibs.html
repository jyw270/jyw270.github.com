Mad Libs Bullet Journaling

Illustrator | HTML | CSS | Javascript

UX / UI

development

2021

Background

This is a school project where we were required to design a simple Mad Libs webpage that takes in inputs from users and then provides them with a simple story. I designed a bullet journaling mad libs because journaling is something I enjoy doing. I addressed the storytelling part by forming the stories in small sections: poems, happenings, important dates, and a note to self.

User Interface

Brainstorming

These are some sketches I did to brainstorm some ideas. I thought of themes that could result in a story: text messaging could form a conversational story, recipe ingredients and instructions could form a fun narrative, and fortune telling could produce a fun little fortune story. But these ideas didn't seem unique enough.

After a few days of contemplating what creative theme I could do, I finally thought of bullet journaling. However, I quickly realized that bullet journaling doesn't include storytelling, so I decided to form the stories in smaller sections.

Digital Sketches

I designed two different interfaces for my mad libs game. The first interface includes the mad libs form for users to fill out and the second page presents the journal with the user's inputs.

Inspired by my favorite childhood book, "Chicken Soup with Rice" by Maurice Sendak, I planned to write a unique poem for each month. There would be 8 lines per poem and every other line would rhyme. To fill the rest of the journal, I looked up several bullet journaling ideas on Pinterest. I wasn't too satisfied with how the right side of the journal looked: the color of the envelope and letter didn't seem cohesive with the rest of the design. I also felt like it was losing the storytelling purpose of the game.

I imagine bullet journaling to resemble a planner, so I added a calendar and a section for important dates to the right side of the journal. I only kept the note to self section because I felt that it had the potential for a short story.

Challenges

We were only given one week to complete this project, so it was a challenge for me to include everything I had planned. Changing the overall design for all 12 months: writing unique poems, changing the calendar dates and important events, and exporting each design was quite tedious and time consuming, but I was able to make it work. In Illustrator, I organized my layers based on each month's theme and text content and toggled with the visibility and lock tool. Then, I went ahead and exported each design as an svg file because I wanted my graphics to look as clear as possible.

Development

This webpage was developed using HTML, CSS, and vanilla JavaScript. It is specifically designed at a 1366 x 768 viewport as that is the current most popular website dimension.

Challenges

When coding this project, I mainly struggled with the form page. I couldn't figure out how to align the text and its corresponding input in a single line or how to align the inputs vertically. After asking for some advice from a classmate, I was able to figure it out. I placed the text and input tag inside a label tag and float: right on the input tags.

After class ended, I wasn't completely satisfied with my game. The transition from the form to the journal page was very disruptive and sudden, so I figured out how to have the form fade out and the book fade in by setting up classes in CSS, then using JavaScript to get the timing right. I also edited my code to allow users the option to submit the placeholder texts if they don't want to fill out the form. I had trouble trying to get the placeholders to turn black for a quick second when the user hits submit. I tried storing the placeholders in a variable and changing the color in Javascript, but I couldn't get it to work. I thought that my only solution was to set each input value to the placeholder text, but that felt too tedious and inefficient. I asked a design professor for advice and ended up using a class to set up and change the color with CSS and JavaScript.

What Can Be Improved

For further development, I could allow users to submit the placeholder texts for any inputs they don't fill out, instead of only enabling that option when they submit a blank form.

Given the time constraint on this project, I am most proud of being able to complete 12 unique designs for each month. I learned when CSS and JavaScript work together, it makes ideas that may seem complex to be a lot simpler.
